# 站点策略

## 定位

本站是一份个人技术刊物，记录软件工程、开发工具与技术表达中的具体问题，以及经过实践后形成的判断。

内容不是按“作者最近做了什么”组织，而是优先回答：

1. 读者正在解决什么问题？
2. 最短可执行路径是什么？
3. 有哪些选择边界和风险？
4. 如何验证结果？

## 核心内容

站点聚焦三个内容方向：

- 软件工程与内容站建设。
- 开发工具与系统问题排查。
- 技术图表与方案表达。

与核心方向相关度低、缺少独立判断或已经失效的内容不再保留。

## 内容价值标准

新内容至少满足以下两项，否则不发布：

- 给出可复用的方法、模型、清单或代码。
- 包含实际操作、验证结果或成果证据。
- 对多个方案进行有边界的比较。
- 补充现有资料中缺失的中文解释。
- 对既有文章形成实质更新或纠错。

## 用户路径

主要用户路径为：

```text
搜索或分享进入文章
  -> 获得明确结论
  -> 通过相关文章或归档继续阅读
  -> 订阅 RSS 或邮件更新
  -> 反馈内容是否有效
```

首页表达本站主张并展示文章；文章页帮助从搜索和外部链接进入的读者快速理解结论、过程和边界。

## 可信度表达

用以下证据帮助读者判断内容是否可信：

- 已完成的开源项目、代码和配置。
- 可运行的示例、图表和操作结果。
- 经过持续更新的代表性文章。
- 对真实问题的复盘和验证记录。

避免抽象能力声明；优先提供可运行代码、实际配置、验证结果和明确适用边界。

## 阶段目标

### 当前阶段：建立可信内容基础

- 统一文章元数据和内容结构。
- 保持文章结构和元数据一致。
- 完善文章发现、订阅、分享和反馈路径。
- 通过自动检查避免内容契约退化。

### 下一阶段：形成专业内容资产

- 每个内容方向形成持续维护的文章。
- 首页保持单层文章入口。
- 根据反馈和搜索需求维护旧文章。
- 当内容数量足够时，再整理专题或手册。
