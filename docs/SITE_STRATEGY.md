# 站点策略

## 定位

本站是 NESNILNEHC 的个人专业站点，围绕擅长领域展示可复用的认知、经过验证的实践和可检查的成果。

内容不是按“最近发生了什么”组织，而是优先回答：

1. 这个问题为什么值得解决？
2. 有哪些可选方法，边界分别是什么？
3. 实际采用了什么方法，结果如何？
4. 读者能否复用其中的判断、步骤或工具？

## 核心内容

站点聚焦以下三个方向：

- 软件工程：架构、工程实践、开发流程与质量保障。
- 开发工具：工具选型、效率改进和问题排查。
- 技术表达：技术写作、图表、文档和知识组织。

与核心方向相关度较低的内容可以保留在归档中，但不进入首页重点推荐和主题导航。

## 内容价值标准

新内容至少满足以下两项：

- 给出可复用的方法、模型、清单或代码。
- 包含实际操作、验证结果或成果证据。
- 对多个方案进行有边界的比较。
- 补充现有资料中缺失的中文解释。
- 对既有文章形成更新、纠错或系列化补充。

## 用户路径

主要用户路径为：

```text
搜索或分享进入文章
  -> 获得明确结论
  -> 通过主题或系列继续阅读
  -> 订阅 RSS 或系列更新
  -> 反馈内容是否有效
```

首页服务于再次访问者和希望了解作者专业方向的读者；文章页服务于从搜索和外部链接直接进入的读者。

## 成果表达

优先展示以下形式的成果：

- 已完成的开源项目、代码和配置。
- 可运行的示例、图表和操作结果。
- 经过更新维护的系列文章。
- 对真实问题的复盘和验证记录。

避免只陈述“熟悉什么”，尽量链接到“做过什么”和“形成了什么判断”。

## 阶段目标

### 当前阶段：建立可信内容基础

- 统一文章元数据和内容结构。
- 将零散文章组织为主题和系列。
- 完善文章发现、订阅、分享和反馈路径。
- 通过自动检查避免内容契约退化。

### 下一阶段：形成专业内容资产

- 每个核心主题至少形成一个持续更新的系列。
- 首页展示代表性文章和成果。
- 根据反馈和搜索需求维护旧文章。
- 将成熟系列整理为专题、手册或可下载资料。
